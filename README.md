# Лабораторная работа №4: Учетная система магазина волшебных палочек

## Цель проекта
Разработка системы учета для магазина "Олливандеры" из мира Гарри Поттера для управления запасами, продажами и поставками волшебных палочек.

## Основные задачи
- Создание реляционной базы данных для хранения информации о палочках, покупателях и поставках
- Реализация функционала отслеживания состояния склада
- Разработка интуитивного графического интерфейса для управления данными
- Обеспечение сохранности данных между сеансами работы

## Ключевые особенности

### Функциональность
- **Управление палочками** - учет готовых палочек (древесина + сердцевина)
- **Регистрация покупателей** - ведение базы данных волшебников
- **Контроль поставок** - управление закупками компонентов
- **Отслеживание склада** - мониторинг остатков материалов
- **Полная очистка** - возможность начать работу с чистого листа

### Техническая реализация
- **Трехуровневые ER-диаграммы** - концептуальный, логический, физический уровни
- **Архитектура MVC** - разделение данных, представления и логики
- **SQLite база данных** - надежное хранение информации
- **Swing интерфейс** - удобное управление через графические формы

## Используемые технологии
- **Java Swing** - пользовательский интерфейс
- **SQLite JDBC** - работа с базой данных
- **MVC Pattern** - архитектура приложения
- **ER Modeling** - проектирование структуры базы данных

## Структура проекта
```
src/
├── main/          # Обработка бизнес-логики
├── gui/           # Графический интерфейс (формы и таблицы)
└── database/      # DTO объекты и работа с БД
```

## Структура базы данных
- **Wands** - палочки (древесина, сердцевина, владелец)
- **Wizards** - покупатели-волшебники
- **Supplies** - поставки компонентов
- **Inventory** - складские остатки
